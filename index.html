1. Introduction

Kitabi is a comprehensive, modern library management system designed to streamline the operations of libraries and educational institutions. The platform provides a complete digital solution for managing book borrowing, hall bookings, book acquisition suggestions, user management, and administrative operations.

Built with a bilingual (English/Arabic) interface and real-time notification system, Kitabi offers both patrons and administrators an intuitive, feature-rich experience. The system is designed for scalability, security, and ease of use, making it suitable for libraries of all sizes.

2. Course Project Requirements

2.1 Overview

This project was developed as part of the Software Engineering Design course at The Hashemite University for the First Semester 2025-2026. The project requirements included creating a mobile/web application for a library system with an excellent interface.

2.2 Required Functions

The system must include the following core functions:

  • Search for a book
  • Book an existing book (reservation)
  • Suggest a book to be bought by the library
  • Search for a suitable hall to make an event
  • Book a time and a hall

3. System Overview

3.1 System Purpose

Kitabi serves as a comprehensive digital solution for library management, bridging the gap between traditional library operations and modern technology. The system aims to provide an efficient, user-friendly platform that benefits both library patrons and administrators.

3.2 Key Features

  • Comprehensive book management with real-time inventory tracking
  • Intelligent hall booking system with conflict prevention
  • Community engagement through book suggestion workflow
  • Real-time notifications via Socket.IO and email
  • Full bilingual support (English/Arabic) with RTL layout
  • Powerful admin dashboard with analytics and reports
  • Complete activity logging and audit trails
  • Role-based access control (User/Admin)

3.3 Target Users

Students

Browse books, make reservations, book halls, suggest books

Faculty Members

Access library resources, book halls for events

Librarians/Admins

Manage entire library operations, generate reports

4. Functional Requirements

4.1 User Management & Authentication

  • User registration with email verification
  • Secure login with JWT authentication
  • Password reset via OTP (6-digit code) sent via email
  • Role-based access control (User/Admin)
  • User profile management
  • Session persistence with auto-logout on token expiry

4.2 Book Catalog & Borrowing System

  • Search books by title, author, or category
  • View book details with bilingual titles/authors/descriptions
  • Reserve books with specified reservation and return dates
  • Real-time inventory tracking (available vs total copies)
  • View personal reservation history with status tracking
  • Return books with automatic inventory updates
  • Overdue detection with automatic status updates
  • Status tracking: Pending, Active, Completed, Cancelled, Overdue
  • Prevent double-booking of unavailable books

4.3 Hall Booking System

  • Search available halls by capacity and time availability
  • View hall details including amenities and bilingual names/descriptions
  • Book halls for events with event name, time range, and purpose
  • Real-time conflict detection to prevent double-bookings
  • View personal booking history
  • Status management: Pending, Confirmed, Cancelled
  • Filter by minimum capacity requirements

4.4 Book Suggestion System

  • Submit book suggestions with title, author, and reason
  • Track suggestion status (Pending, Approved, Rejected, Purchased)
  • View personal suggestion history
  • Admin review workflow with status updates and notes

4.5 Real-time Notification System

  • Real-time push notifications via Socket.IO
  • Desktop notifications (browser-based with permission)
  • Toast notifications with priority levels (Low, Normal, High, Urgent)
  • Bilingual notification messages
  • Mark as read/unread functionality
  • Delete notifications and track unread count

4.6 Admin Dashboard

  • Comprehensive overview with key statistics
  • Total books, users, active reservations, overdue items
  • Recent activity feed
  • Quick access to all management sections

4.7 Reports & Analytics

  • Dashboard Summary: Quick stats for all resources
  • Most Borrowed Books: Ranking with date range filters
  • Inventory Report: Total/available/borrowed copies per book
  • Category Statistics: Distribution of books by category
  • Monthly Activity: Trend analysis for reservations and bookings
  • Overdue Summary: List of overdue reservations with days overdue
  • User Activity Reports: Per-user borrowing and booking history

5. Non-Functional Requirements

5.1 Security

  • JWT-based authentication with secure token handling
  • Password hashing using bcryptjs (salt round 10)
  • Protected routes with authentication middleware
  • Role-based authorization (admin middleware)
  • Input validation using Zod schemas
  • XSS and SQL injection prevention via ORM
  • Activity logging for audit trails

5.2 Performance

  • PostgreSQL database with indexed queries
  • React Query caching for optimized API calls
  • Real-time updates via Socket.IO (bidirectional)
  • Vite for fast development and optimized production builds

5.3 Usability

  • Intuitive, modern UI with gradient designs
  • Responsive design (mobile-first approach)
  • Loading states and user-friendly error messages
  • Accessibility considerations (semantic HTML, ARIA labels)

5.4 Reliability

  • Error handling at all layers
  • Graceful degradation for failed requests
  • Database constraints to maintain data integrity
  • Email verification to ensure valid user accounts

9. Core Use Cases

9.1 Student Book Borrowing Workflow

  1. Student logs into the system
  2. Searches for books by title, author, or category
  3. Views book details and checks availability
  4. Reserves book with specified dates
  5. Receives real-time notification of successful reservation
  6. Borrows book from library
  7. Returns book before due date
  8. System automatically updates inventory and status

9.2 Hall Booking for Events

  1. User searches available halls by capacity and date
  2. Views hall details including amenities
  3. Submits booking request with event details
  4. System checks for booking conflicts
  5. Admin reviews and confirms booking
  6. User receives confirmation notification
  7. User uses hall for scheduled event

9.3 Book Suggestion Process

  1. Student submits book suggestion with details
  2. Admin reviews suggestion in admin panel
  3. Admin approves or rejects with notes
  4. Student receives notification of decision
  5. If approved, admin purchases book
  6. Admin marks suggestion as purchased
  7. Admin adds new book to catalog
  8. Student receives final notification

10. Conclusion

Kitabi represents a comprehensive solution for modern library management, successfully combining traditional library operations with cutting-edge web technologies. The system addresses the core requirements specified in the course project while extending functionality to provide a production-ready application.

10.1 Key Achievements

  • Successfully implemented all required functions: book search, reservation, hall booking, and suggestions
  • Created an intuitive, modern interface with excellent user experience
  • Developed a scalable, secure architecture using industry-standard technologies
  • Implemented full bilingual support for diverse user bases
  • Added real-time notifications for enhanced user engagement
  • Created comprehensive admin tools for complete library oversight
  • Implemented detailed analytics and reporting capabilities

10.2 Technical Excellence

The project demonstrates mastery of full-stack development, incorporating modern practices such as TypeScript for type safety, React Query for efficient state management, and Socket.IO for real-time communication. The implementation follows security best practices including JWT authentication, password hashing, and input validation.

10.3 Impact and Benefits

Kitabi streamlines library operations by automating manual processes, reducing administrative overhead, and improving user satisfaction. The system provides:

  • 24/7 access to library services
  • Reduced wait times and improved resource utilization
  • Better inventory management and tracking
  • Data-driven decision making through analytics
  • Enhanced user engagement and satisfaction

10.4 Future Enhancements

While Kitabi is production-ready, potential future enhancements include:

  • Mobile native applications for iOS and Android
  • Integration with university student information systems
  • QR code scanning for quick book checkout
  • AI-powered book recommendations
  • Digital library integration for e-books
  • Advanced analytics with machine learning insights

Final Note

In conclusion, Kitabi successfully meets and exceeds the project requirements, delivering a robust, scalable, and user-friendly library management system that can serve educational institutions of all sizes.
